Opmeetapp BOAB 2025 V30 - Buffer Prioriteit Fix

Probleem

In de huidige implementatie wordt de H-buffer (lichtblauw) niet consequent bovenop de V-buffer (lichtrood) weergegeven in overlappende gebieden.

Oplossing

We hebben een eenvoudige maar effectieve aanpak geïmplementeerd die garandeert dat de H-buffer altijd zichtbaar is boven de V-buffer:

  1. Op mask-niveau: we verwijderen V-buffer op plekken waar H-buffer aanwezig is (in overlappende gebieden)
  2. Bij het tekenen: we tekenen eerst de V-buffer, dan de H-buffer

Deze aanpak zorgt ervoor dat de H-buffer (lichtblauw) altijd zichtbaar is bovenop de V-buffer (lichtrood) in alle overlappende gebieden.

Technische Details

1. Prioriteit op mask-niveau

// BELANGRIJK: Ongeacht policy, H wint ALTIJD bij buffer-tegen-buffer overlaps // Dit is voor de visuele weergave - voor de berekeningen gebruiken we nog steeds de policy // Forceer altijd H-buffer over V-buffer voor consistente kleuren for (let i = 0; i < microRows * microCols; i++) { // Als er overlap is tussen V-buffer en H-buffer, prioriteer H-buffer if (maskV[i] && maskH[i] && !coreV[i] && !coreH[i]) { // Behoud alleen de H-buffer voor de visualisatie maskV[i] = 0; } }

2. Eenvoudigere tekenlogica

// Eenvoudige aanpak: eerst V, dan H - omdat we al prioriteit hebben gegeven in de maskers // met maskV[i] = 0 waar maskH[i] = 1 in overlappende gebieden // STAP 1: Teken V-buffer (laag) if (settings.layerV) { ctx.fillStyle = 'rgba(239,68,68,0.18)'; // lichtrood for (let mr = 0, idx = 0; mr < microRows; mr++) { for (let mc = 0; mc < microCols; mc++, idx++) { if (maskV[idx] && !coreV[idx] && !(settings.layerH && coreH[idx])) { ctx.fillRect( mc * microPxX, mr * microPxY, microPxX + 0.5, // +0.5 voorkomt gaten tussen pixels microPxY + 0.5 ); vBufferCount++; } } } } // STAP 2: Teken H-buffer (hoog) if (settings.layerH) { ctx.fillStyle = 'rgba(59,130,246,0.18)'; // lichtblauw for (let mr = 0, idx = 0; mr < microRows; mr++) { for (let mc = 0; mc < microCols; mc++, idx++) { if (maskH[idx] && !coreH[idx] && !(settings.layerV && coreV[idx])) { ctx.fillRect( mc * microPxX, mr * microPxY, microPxX + 0.5, microPxY + 0.5 ); hBufferCount++; } } } }

Waarom werkt deze aanpak?

Deze aanpak werkt om de volgende redenen:

  1. Prioriteit vroeg toepassen: Door de V-buffer al in de maskfase te verwijderen waar H-buffer aanwezig is, is er geen kans meer op verkeerde overlap.
  2. Eenvoudige tekenlogica: Door de tekenlogica te vereenvoudigen, vermijden we complexe overlappingsberekeningen die tot fouten kunnen leiden.
  3. Consistentie: Deze aanpak werkt consistent in alle situaties, ongeacht de vorm of grootte van de buffers.

Resultaat

Na deze wijzigingen zal de H-buffer (lichtblauw) altijd bovenop de V-buffer (lichtrood) worden weergegeven in alle overlappende gebieden, zoals vereist door de specificatie.
