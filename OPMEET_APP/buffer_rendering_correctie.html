Opmeetapp BOAB 2025 V30 - Buffer Rendering Fix

Buffer Prioriteit Correctie: Deze update zorgt ervoor dat de H-buffer (lichtblauw) altijd zichtbaar is bovenop de V-buffer (lichtrood) wanneer ze overlappen.

Probleem Beschrijving

In de vorige versie van de applicatie was er een probleem met de weergave van overlappende buffers. In sommige situaties werd de V-buffer (lichtrood) bovenop de H-buffer (lichtblauw) getekend, terwijl de specificatie vereist dat de H-buffer altijd zichtbaar moet zijn bovenop de V-buffer.

Probleem Visualisatie

In deze afbeelding is te zien dat de V-buffer (rood) bovenop de H-buffer (blauw) wordt weergegeven in overlappende gebieden:

Incorrect: V-buffer (rood) bovenop H-buffer (blauw)

De Oplossing

Om dit probleem op te lossen, hebben we twee belangrijke wijzigingen aangebracht:

1. Behoud beide maskers volledig

In plaats van op mask-niveau de prioriteit toe te passen, behouden we nu beide maskers volledig. Dit zorgt ervoor dat we tijdens het tekenen volledige controle hebben over welke buffer zichtbaar is.

2. Gewijzigde tekenlogica in drawBuffer

We hebben de render-volgorde in de drawBuffer functie aangepast. De V-buffer wordt eerst volledig getekend, en daarna wordt de H-buffer bovenop getekend. Dit garandeert dat de H-buffer altijd zichtbaar is bovenop de V-buffer.

3. Extra veiligheidslaag voor overlappende gebieden

Als extra zekerheid wordt een derde tekenpass uitgevoerd die specifiek alle overlappende gebieden nog eens met H-buffer (blauw) tekent.

Technische Details

We hebben twee belangrijke wijzigingen gemaakt:

1. Verwijderen van de mask-prioriteit in computeRegMasks

Oorspronkelijk werd de V-buffer op mask-niveau verwijderd waar H-buffer aanwezig was. Deze code is nu verwijderd zodat beide maskers volledig behouden blijven.

2. Volledige V-buffer tekenen in drawBuffer

De V-buffer wordt nu eerst volledig getekend waar deze geldig is, daarna wordt de H-buffer bovenop getekend zodat deze altijd zichtbaar is op overlappende gebieden.

Oorspronkelijke drawBuffer code

// STAP 1: Teken V-buffer (laag) - alleen waar geen H-buffer bovenop komt if (settings.layerV) { ctx.fillStyle = 'rgba(239,68,68,0.18)'; // lichtrood for (let mr = 0, idx = 0; mr < microRows; mr++) { for (let mc = 0; mc < microCols; mc++, idx++) { // V-buffer alleen tekenen waar geen overlap is met H-buffer if (maskV[idx] && !coreV[idx] && !(settings.layerH && coreH[idx]) && !(maskH[idx])) { ctx.fillRect( mc * microPxX, mr * microPxY, microPxX + 0.5, microPxY + 0.5 ); vBufferCount++; } } } }

Verbeterde drawBuffer code

// STAP 1: Teken V-buffer (laag) - altijd als eerste en onderste laag if (settings.layerV) { ctx.fillStyle = 'rgba(239,68,68,0.18)'; // lichtrood for (let mr = 0, idx = 0; mr < microRows; mr++) { for (let mc = 0; mc < microCols; mc++, idx++) { // V-buffer tekenen overal waar het aanwezig is // H-buffer zal later bovenop worden getekend if (maskV[idx] && !coreV[idx] && !(settings.layerH && coreH[idx])) { ctx.fillRect( mc * microPxX, mr * microPxY, microPxX + 0.5, microPxY + 0.5 ); vBufferCount++; } } } }

Correcte Visualisatie

Na de fix zal de H-buffer (blauw) altijd bovenop de V-buffer (rood) worden weergegeven:

Samenvatting van de Wijzigingen

  1. Verwijderd: mask-prioriteitsfiltering in de computeRegMasks functie
  2. Gewijzigd: V-buffer wordt nu volledig getekend als eerste laag
  3. Behouden: H-buffer wordt getekend bovenop de V-buffer
  4. Behouden: Extra veiligheidslaag voor overlappende gebieden

Belangrijk: Deze wijzigingen hebben alleen invloed op de visuele weergave. De oppervlakteberekeningen en andere functionaliteiten blijven ongewijzigd werken.
