Opmeetapp BOAB 2025 V30 - V-buffer Prioriteit Fix

Gecorrigeerd Probleem

In de vorige implementatie werd de H-buffer (lichtblauw) bovenop de V-buffer (lichtrood) weergegeven in overlappende gebieden. Volgens de nieuwe specificatie moet dit echter omgedraaid worden: de V-buffer moet voorrang krijgen op de H-buffer.

Nieuwe Oplossing

We hebben de prioriteit omgedraaid zodat de V-buffer (lichtrood) nu altijd zichtbaar is bovenop de H-buffer (lichtblauw):

  1. Op mask-niveau: we verwijderen H-buffer op plekken waar V-buffer aanwezig is (in overlappende gebieden)
  2. Bij het tekenen: we tekenen eerst de H-buffer, dan de V-buffer

Deze aanpak zorgt ervoor dat de V-buffer (lichtrood) altijd zichtbaar is bovenop de H-buffer (lichtblauw) in alle overlappende gebieden.

Technische Details

1. Prioriteit op mask-niveau: V wint van H

// BELANGRIJK: Ongeacht policy, V wint ALTIJD bij buffer-tegen-buffer overlaps // Dit is voor de visuele weergave - voor de berekeningen gebruiken we nog steeds de policy // Forceer altijd V-buffer over H-buffer voor consistente kleuren for (let i = 0; i < microRows * microCols; i++) { // Als er overlap is tussen V-buffer en H-buffer, prioriteer V-buffer if (maskV[i] && maskH[i] && !coreV[i] && !coreH[i]) { // Behoud alleen de V-buffer voor de visualisatie maskH[i] = 0; } }

2. Gewijzigde tekenlogica: eerst H, dan V

// Eenvoudige aanpak: eerst H, dan V - omdat we al prioriteit hebben gegeven in de maskers // met maskH[i] = 0 waar maskV[i] = 1 in overlappende gebieden // STAP 1: Teken H-buffer (laag) if (settings.layerH) { ctx.fillStyle = 'rgba(59,130,246,0.18)'; // lichtblauw for (let mr = 0, idx = 0; mr < microRows; mr++) { for (let mc = 0; mc < microCols; mc++, idx++) { if (maskH[idx] && !coreH[idx] && !(settings.layerV && coreV[idx])) { ctx.fillRect( mc * microPxX, mr * microPxY, microPxX + 0.5, microPxY + 0.5 ); hBufferCount++; } } } } // STAP 2: Teken V-buffer (hoog) if (settings.layerV) { ctx.fillStyle = 'rgba(239,68,68,0.18)'; // lichtrood for (let mr = 0, idx = 0; mr < microRows; mr++) { for (let mc = 0; mc < microCols; mc++, idx++) { if (maskV[idx] && !coreV[idx] && !(settings.layerH && coreH[idx])) { ctx.fillRect( mc * microPxX, mr * microPxY, microPxX + 0.5, microPxY + 0.5 ); vBufferCount++; } } } }

Belangrijk

De tussenzone tussen V-offset en H-offset blijft nu deel van de H-strook, zoals vereist. Dit is omdat we alleen de overlappende gebieden wijzigen, en de rest van de H-buffer intact laten.

Waarom werkt deze aanpak?

Deze aanpak werkt om de volgende redenen:

  1. Omgekeerde prioriteit: Door de H-buffer op mask-niveau te verwijderen waar V-buffer aanwezig is, zorgen we ervoor dat de V-buffer voorrang krijgt.
  2. Tekenlogica aangepast: Door eerst H en dan V te tekenen, garanderen we dat V altijd bovenop H zichtbaar is.
  3. Behoud tussenzone: Omdat we alleen ingrijpen bij overlappende cellen (waar zowel maskV als maskH actief zijn), blijven andere delen van beide buffers intact.

Resultaat

Na deze wijzigingen zal de V-buffer (lichtrood) altijd bovenop de H-buffer (lichtblauw) worden weergegeven in alle overlappende gebieden, en de tussenzone tussen beide offsets blijft deel van de H-strook, zoals vereist volgens de specificatie.
