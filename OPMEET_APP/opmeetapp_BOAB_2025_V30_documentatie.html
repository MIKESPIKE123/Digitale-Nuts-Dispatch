Buffer Visualisatie Correctie

Deze update corrigeert de weergave van overlappende buffers, zodat de H-buffer (lichtblauw) altijd correct bovenop de V-buffer (lichtrood) wordt weergegeven.

Voor correctie

De V-buffer (rood) werd soms boven de H-buffer (blauw) weergegeven in overlappende gebieden.

V-buffer (rood) incorrect bovenop H-buffer (blauw)

Na correctie

De H-buffer (blauw) wordt nu altijd bovenop de V-buffer (rood) weergegeven in overlappende gebieden.

H-buffer (blauw) correct bovenop V-buffer (rood)

Wat is er veranderd?

Bewaren van beide maskers

Voorheen werd de V-buffer op mask-niveau al verwijderd waar de H-buffer aanwezig was, waardoor we controle verloren over de volgorde van tekenen. Nu bewaren we beide maskers volledig en bepalen de render-volgorde tijdens het tekenen.

// Verwijderde code (vorige versie): for (let i = 0; i < microRows * microCols; i++) { if (maskV[i] && maskH[i] && !coreV[i] && !coreH[i]) { maskV[i] = 0; // Verwijderde V-buffer op mask-niveau } }

Verbeterde tekenlogica

We tekenen nu eerst de complete V-buffer als onderlaag, daarna de H-buffer als bovenlaag. Dit zorgt ervoor dat de H-buffer altijd zichtbaar is boven de V-buffer.

// Nieuwe code: // STAP 1: Teken V-buffer (laag) - altijd als eerste en onderste laag if (settings.layerV) { ctx.fillStyle = 'rgba(239,68,68,0.18)'; // lichtrood for (let mr = 0, idx = 0; mr < microRows; mr++) { for (let mc = 0; mc < microCols; mc++, idx++) { if (maskV[idx] && !coreV[idx] && !(settings.layerH && coreH[idx])) { ctx.fillRect( mc * microPxX, mr * microPxY, microPxX + 0.5, microPxY + 0.5 ); vBufferCount++; } } } }

Vereenvoudigde overlap-check

De extra veiligheidscheck voor overlappende gebieden is nu directer en eenvoudiger, wat de leesbaarheid verbetert en de prestaties kan verhogen.

// Nieuwe, vereenvoudigde overlap-check: if (settings.layerH && settings.layerV) { ctx.fillStyle = 'rgba(59,130,246,0.18)'; // lichtblauw (H wint) for (let mr = 0, idx = 0; mr < microRows; mr++) { for (let mc = 0; mc < microCols; mc++, idx++) { // Controleert direct op echte overlaps if (maskH[idx] && maskV[idx] && !coreH[idx] && !coreV[idx]) { ctx.fillRect( mc * microPxX, mr * microPxY, microPxX + 0.5, microPxY + 0.5 ); } } } }

Technisch Detail

Deze wijziging behoudt alle oppervlakteberekeningen zoals ze waren, maar zorgt voor een consistente visuele weergave. De correctie is minimaal invasief en behoudt alle bestaande functionaliteit.

Voordelen van deze aanpak

  • Consistente visualisatie waarbij H-buffer altijd bovenop V-buffer wordt weergegeven
  • Geen impact op oppervlakteberekeningen of andere functionaliteiten
  • Duidelijkere code die het tekenproces expliciet maakt
  • Betere prestaties door vereenvoudiging van de overlap-detectie
